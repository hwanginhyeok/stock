# 야간작업 — 2026-04-06 (stock 프로젝트)

## 작업 1: InvestOS 이벤트 체계 대개편 (1-35)

### 1-1. EventType geo/stock 분리
- `src/core/models.py`의 EventType enum 확장
  - Geo용: diplomatic, military, sanctions, energy, trade, territorial
  - Stock용: earnings, analyst, product, regulatory, macro, deal, sector
- `src/core/database.py` OntologyEventDB에 반영
- `deep_analysis.py` 프롬프트에서 카테고리별 다른 event_type 사용하도록 수정

### 1-2. 이벤트 스토리 체이닝
- OntologyEvent 모델에 `story_thread` 필드 추가 (문자열, 같은 스토리라인 이벤트를 묶는 키)
  - 예: "이란_호르무즈_봉쇄", "테슬라_FSD_승인"
- DB 마이그레이션
- `deep_analysis.py`에서 Ollama에 기존 이벤트 목록을 주고, 새 이벤트가 기존 스토리에 이어지는지 판단하도록 프롬프트 수정
- 같은 story_thread 이벤트끼리 시간순 정렬 → 타임라인 뷰 가능

### 1-3. 과거 이벤트 백필
- 이슈별로 뉴스 DB에서 최근 7일 데이터를 가져와서 Ollama로 핵심 이벤트 추출
- 기존 53개 이벤트와 중복 체크 후 추가
- story_thread로 연결

### 1-4. InvestOS UI 이벤트 타임라인 뷰
- `/api/issues/{id}/events` 엔드포인트에서 story_thread별 그룹핑 추가
- `static/index.html`에서 이벤트를 story_thread별로 묶어서 타임라인 형태로 표시
- 이벤트 카드에 severity 색상 표시 (critical=빨강, major=주황, moderate=파랑, minor=회색)

## 작업 2: 관계도 복잡성 해소 (1-36)

### 2-1. 대안 조사
- 현재 문제: 엔티티 2,300+개, 관계 3,900+개 → D3 force graph가 너무 복잡
- 조사할 대안:
  a) **Top N 필터**: 뉴스 언급 빈도/관계 수 기준 상위 10~20개만 표시
  b) **클러스터링**: 엔티티를 그룹으로 묶어서 펼치기/접기
  c) **레이어 뷰**: 엔티티 타입별 레이어로 분리 (국가/기업/인물/자산)
  d) **관계 설명 개선**: 툴팁이 아닌 사이드패널에 관계 상세 + 근거 뉴스

### 2-2. 구현 (가장 효과 큰 것 우선)
- Top10 필터 구현이 가장 빠르고 효과 큼
  - `/api/issues/{id}/graph` 엔드포인트에 `?top=10` 파라미터 추가
  - 뉴스 빈도 또는 관계 degree 기준 상위 N개 엔티티 + 그들 간 관계만 반환
  - UI에 슬라이더 또는 드롭다운 (Top 5 / 10 / 20 / All)
- 관계 설명 UX 개선
  - 관계선 클릭 시 사이드패널에 evidence + 관련 뉴스 표시
  - 관계 타입별 색상 구분

### 2-3. 브리핑 작성
- 작업 완료 후 `docs/프로젝트/overnight_brief_20260407.md`에 결과 기록
- 변경 사항, 스크린샷 설명, 남은 이슈 정리
