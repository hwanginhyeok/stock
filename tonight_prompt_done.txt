## 기능 A: 1-41 — 엔티티 노이즈 필터 강화 [단순]

### Phase 1: CEO 리뷰
- **문제**: Ollama가 "40억 달러", "$2.9B", "E-9 visa" 등을 엔티티로 추출
- **스코프**: review_entities.py + 엔티티 추출/분류 코드

### Phase 2: 구현
1. 현재 엔티티 DB에서 노이즈 패턴 분석
2. 정규식 필터 추가:
   - 금액: `$숫자`, `숫자억/조/만`, `$X.XB/M/K`
   - 비율: `X%`, `X배`, `X배수`
   - 코드: `E-9`, `H-1B` 등 비자/규격 코드
3. review_entities.py에 필터 적용
4. 기존 DB 노이즈 엔티티 정리

### Phase 3: 테스트
- false positive 확인 (정상 엔티티 삭제 방지)
- 필터 전/후 엔티티 수 비교

### Phase 4: 코드리뷰
**CRITICAL**: false positive 방지
**INFO**: 패턴 확장성

## 기능 D: 1-38 — 관계도 깊이/밀도 제어 [단순]

### Phase 1: CEO 리뷰
- **문제**: 1차~3차 관계 전부 표시 → 과다. 깊이 필터 필요
- **스코프**: 관계도 생성/조회 코드

### Phase 2: 구현

3. API/UI에 깊이 선택 옵션
4. 기본값: 2차까지

### Phase 3: 테스트
- 깊이별 노드 수 비교
- pruning 후 핵심 관계 누락 없는지

### Phase 4: 코드리뷰
**CRITICAL**: 핵심 관계 pruning 방지

## 기능 F: 1-39 — Stock 엔티티 온톨로지 설계 [복잡 → /hih-ontology 적용]

### Phase 1: CEO 리뷰
- **의존**: E(5-10) 완료 후 착수
- **문제**: 현재 엔티티가 타입 라벨만 있음. "왜 존재하는가"의 속성 체계 없음
- **참조**: GEO 엔티티는 행위자별 목표·전략·달성여부가 핵심

### Phase 2: 구현 (/hih-ontology 프레임워크)
1. **개체 식별**: Stock 도메인 핵심 엔티티 유형
   - 기업: 비즈니스모델, 경쟁우위, 밸류에이션, 리스크
   - 중앙은행: 정책목표, 수단, 달성여부
   - 지표: 현재값, 추세, 시그널
2. **속성 정의**: 각 유형의 본질적 속성 (왜 이 엔티티가 중요한가)
3. **관계 매핑**: 엔티티 간 인과/영향 관계
4. **제약조건**: 시계열 변화, 불확실성 표현
5. 추출 프롬프트 분리 (Ollama용)

### Phase 3: 테스트
- 기존 엔티티 DB를 새 스키마로 매핑 가능한지 검증
- 추출 프롬프트 테스트 (샘플 뉴스 3개)

### Phase 4: 코드리뷰
**CRITICAL**: 기존 시스템과 호환성, 마이그레이션 경로
**INFO**: 스키마 확장성
