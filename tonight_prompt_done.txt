## 규칙
- 주석/커밋 메시지 한국어
- 기존 코드 패턴 따르기
- 각 작업 완료마다 커밋
- TASK.md 갱신 필수
- 각 기능마다 4단계 파이프라인 필수: CEO리뷰 → 구현 → 테스트 → 코드리뷰

## 기능 A: 모닝 이메일 Graceful Degradation

### Phase 1: CEO 리뷰
- **문제**: 외부 API(FRED, 환율, 증권) 하나라도 장애나면 전체 모닝 이메일 발송 실패
- **현재**: cron 09:00 실행 → API 장애 시 사용자는 이메일 못 받음
- **목표**: 부분 실패 시에도 가용 데이터로 이메일 발송. "데이터 누락" 표시로 투명성 유지.
- **스코프**: send_morning_email.py + email_publisher.py
- **판정**: P1. 서비스 안정성 핵심. 구현 비용 낮고 효과 큼.

### Phase 2: 구현
1. `send_morning_email.py` 읽기 — 데이터 수집 흐름 파악
2. `src/publishers/email_publisher.py` 읽기 — 이메일 렌더링 흐름
3. `src/collectors/` 각 수집기 확인
4. 수정:
   - 각 데이터 수집을 개별 try-except로 감싸기
   - 실패한 섹션 → `{"error": "수집 실패", "source": "FRED"}` 형태로 대체
   - 이메일 템플릿에 실패 섹션 표시 로직 추가 (조건부 렌더링)
   - 이메일 제목: 실패 있으면 "[일부 누락]" 접두사
   - 전체 실패(모든 수집 실패) 시에만 이메일 발송 스킵
5. 구문 검증 + 커밋

### Phase 3: 테스트
`tests/test_email_graceful.py` 작성:
- 전체 정상 → 정상 이메일
- FRED만 실패 → 이메일 발송 + FRED 섹션 "데이터 누락" 표시
- 전체 실패 → 이메일 미발송
- 이메일 제목 접두사 테스트

### Phase 4: 코드리뷰 (2-pass)
**Pass 1 CRITICAL**: 예외 삼킴(except: pass 패턴), 시크릿 로깅, 이메일 수신자 하드코딩
**Pass 2 INFO**: 에러 메시지 일관성, 로깅 레벨
발견사항 있으면 즉시 수정.

## 기능 B: RSS 수집기 안정성 + 재시도

### Phase 1: CEO 리뷰
- **문제**: 최근 RSS 소스 추가(연합뉴스, 한국경제). 개별 소스 파싱 실패 시 전체 수집 중단 가능성.
- **영향**: 뉴스 타임라인 데이터 누락 → 분석 품질 저하
- **스코프**: RSS 수집 관련 코드
- **판정**: P2. 기능 A와 같은 맥락(부분 실패 허용).

### Phase 2: 구현
1. RSS 수집 코드 찾기 (최근 커밋 `cc079d1`, `cb31ddc` 참고)
2. 개별 소스 실패 격리 확인/보강:
   - 소스별 try-except
   - 실패 소스 로깅 (소스명 + 에러 메시지)
   - 실패 소스 카운터 → 전체 결과에 포함
3. 네트워크 타임아웃 설정 확인 (requests.get timeout 파라미터)
4. 구문 검증 + 커밋

### Phase 3: 테스트
`tests/test_rss_collector.py` 작성:
- 정상 RSS 응답 → 파싱 성공
- 비정상 XML → 해당 소스만 스킵
- 타임아웃 → 해당 소스만 스킵
- 전체 소스 실패 → 빈 결과 반환 (크래시 아님)

### Phase 4: 코드리뷰
**Pass 1 CRITICAL**: 무한 재시도 루프, 메모리 누수(대용량 응답), URL 검증
**Pass 2 INFO**: 중복 코드, 로깅

## 기능 C: 데이터 수집기 단위 테스트

### Phase 1: CEO 리뷰
- **문제**: collectors/ 모듈에 테스트가 부족. 외부 API 변경 시 발견이 늦어짐.
- **판정**: 테스트 부채. 모킹 기반으로 파싱 로직만 검증.

### Phase 2: 구현
`tests/test_collectors.py` 작성:
- `src/collectors/` 내 각 수집기 import
- requests.get을 모킹하여 실제 API 호출 없이 파싱 로직만 테스트
- 케이스: 정상 응답, 빈 응답, 비정상 JSON, 타임아웃

### Phase 3: 테스트 실행
`pytest tests/test_collectors.py -v`

### Phase 4: 코드리뷰
테스트 코드 자체 리뷰: 모킹 정확성, assertion 충분성
