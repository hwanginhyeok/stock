# 아티클 리서치 스킬

## 목적
- 투자 아티클 작성을 위한 체계적 리서치 프로세스 정의
- 직감 → 검증 → 뒷받침 → 시각화의 4단계 워크플로우

## 실행 조건
- 새로운 투자 아티클 기획 시
- 기존 아이디어를 데이터로 검증할 때

## 절차

### Phase 1: 직감 정리 (Intuition Capture)
1. 아이디어의 핵심 테제를 1~2문장으로 정리
2. 비유/유추 구조 정리 (A:B = C:D 형태)
3. `data/articles/{순번}_{티커}_{slug}/` 디렉토리 생성 (아래 네이밍 규칙 참조)
4. 아이디어 메모를 `research/thesis.md`에 기록

### Phase 2: 데이터 검증 (Data Verification)
1. 테제의 각 주장을 분해 (claim 단위)
2. 각 claim에 대해:
   - 뒷받침하는 데이터 검색 (뉴스, 실적, 시장 점유율 등)
   - 반박하는 데이터도 함께 수집 (균형 잡힌 분석)
3. 검증 결과를 `research/claims.md`에 기록
4. 출처는 반드시 URL + 날짜 포함

### Phase 3: 뒷받침 자료 선정 (Supporting Evidence)
1. 검증된 claim 중 아티클에 사용할 핵심 데이터 선별
2. 자료 유형별 분류:
   - 📊 정량 데이터: 매출, 점유율, 주가, 밸류에이션
   - 📰 정성 데이터: 전문가 의견, 업계 동향, 전략 분석
   - 📈 시계열 데이터: 트렌드, 성장률, 비교 차트용
3. 수집한 원본 자료는 `research/` 하위에 저장

### Phase 4: 시각화 계획 (Visualization Plan)
1. 아티클 스토리라인에 맞는 차트/그래픽 목록 작성
2. 차트 유형 결정:
   - 타임라인: 연도별 이벤트 흐름
   - 달성 현황 테이블: 목표별 상태 색상 코딩
   - 카드: 핵심 개념 비교 (2×2)
   - 바 차트: 수치 비교 (gap, 성장률)
3. 시각화는 `generate-visuals` 스킬에 따라 생성 → `visuals/`에 저장

## 디렉토리 구조

```
data/articles/{순번}_{티커}_{slug}/
├── research/        # 리서치 노트 (raw + verified 통합)
├── drafts/          # 초안
├── visuals/         # 생성된 PNG 이미지
├── published/       # 게시 완료본
└── generate_visuals.py  # 시각화 생성 스크립트
```

## 네이밍 규칙
- 순번: 3자리 (`001`, `002`, ...)
- 티커: 대문자 (`TSLA`, `PLTR`, `NVDA` 등)
- slug: 한국어 kebab-case
- 예: `001_TSLA_마스터플랜-20년의-논리`

## 산출물 (X 롱폼 아티클 기준)
| 항목 | 설명 |
|------|------|
| 포맷 | X 아티클(Notes) — 제목 있는 롱폼 |
| 방향 | 스레드로 잘게 나누지 않음. 하나의 글에 깊이 |
| 구조 | 논리를 쌓는 스토리텔링 — 독자가 결론에 도달하게 |
| 시각화 | PNG 이미지 첨부 (타임라인, 테이블, 카드 등) |
| 소개 | 아티클 링크를 짧은 트윗으로 별도 소개 가능 |

## 주의사항
- 투자 권유/추천 표현 금지 — 분석과 정보 전달만
- 면책조항 반드시 포함
- 출처 명시 필수
- 데이터는 최신 기준 (최소 최근 1분기 이내)
- 반박 가능한 논리에 대한 counter-argument도 언급
