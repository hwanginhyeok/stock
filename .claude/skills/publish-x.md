# X 게시 스킬

## 목적
- 완성된 아티클을 X Notes에 게시하는 표준 절차
- 매 게시마다 동일한 품질과 순서를 보장

## 실행 조건
- `published/{회차}_{티커}_{slug}/` 폴더에 md + 이미지 파일이 준비된 경우

---

## 게시 체크리스트

```
□ published/{회차}/ 폴더 확인
  □ *_x_publish.md
  □ X_IMAGE_GUIDE.md (📎 번호 → 소스 파일 매핑)
  □ visuals/ + images/ 폴더

□ X_IMAGE_GUIDE.md 열어서 이미지 순서 확인
  □ 📎 1 → 어떤 파일?
  □ 📎 2 → 어떤 파일?
  □ ... (파일명 번호 ≠ 삽입 순서일 수 있음)

□ X Notes 새 아티클 생성
  □ 제목 입력
  □ x_publish.md 내용 붙여넣기
  □ 📎 마커 위치에 X_IMAGE_GUIDE.md 순서대로 이미지 삽입
  □ 시각적 호흡 확인 — 흐름 전환·인용("") 앞뒤 빈 줄 유지됐는지
  □ 면책조항 확인 (글 마지막)

□ 소개 트윗 작성
  □ 핵심 훅 1문장 (아티클 첫 문장 or 별도)
  □ 주제 해시태그 포함 (아래 참조)
  □ 아티클 링크 포함

□ 발행 확인
  □ 모바일에서 이미지 렌더링 확인
  □ TASK.md 상태 → 완료
```

---

## 주제별 해시태그

| 주제 | 권장 해시태그 |
|------|--------------|
| 테슬라 | `#테슬라` `#TSLA` `#Tesla` |
| 팔란티어 | `#팔란티어` `#PLTR` `#Palantir` |
| 미국주식 | `#미국주식` `#해외주식` |
| 한국주식 | `#한국주식` `#국내주식` |
| 코인 | `#비트코인` `#이더리움` `#코인` (종목별 추가) |

---

## 소개 트윗 패턴

```
{핵심 훅 — 독자가 멈추게 만드는 한 줄}

전체 아티클 →
{X Notes 링크}
```

**좋은 훅 예시:**
- "테슬라를 자동차 회사로 보는 한, 절대 이해할 수 없는 것들이 있다."
- 아티클 제목을 그대로 쓰거나, 첫 문장 활용

---

## 주의사항
- X Notes는 API 미지원 — 반드시 수동 업로드
- 이미지는 📎 마커 순서대로 삽입 (타임라인 → 달성현황 → 카드)
- 면책조항 누락 금지
