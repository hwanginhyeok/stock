# 이미지 마커 배치 & 검증 (image-markers)

## 목적

- 아티클 본문에 📎 이미지 마커를 체계적으로 배치한다
- 마커 번호 ↔ 실제 파일 매핑의 정합성을 보장한다
- v2 / x_publish / naver HTML 간 마커 수 불일치를 방지한다

## 실행 조건

| 트리거 | 설명 |
|--------|------|
| v1 초안 작성 시 | article-drafting 3단계에서 📎 마커 배치 |
| 시각화 생성 완료 후 | generate-visuals 후 마커 ↔ 파일 매핑 검증 |
| 퇴고 시 | article-revision에서 마커 위치 재검토 |
| 네이버 변환 시 | naver-packaging에서 최종 교차 검증 |

## 마커 문법

```markdown
📎 *[이미지: {설명}]*
```

- `📎` 이모지 + 이탤릭 대괄호 안에 설명
- 설명은 독자가 아닌 **작성자를 위한 레이블** (게시 시 삭제됨)
- 예: `📎 *[이미지: HBM 세대별 대역폭 비교]*`

## 절차

### 1단계: 마커 배치 (v1 작성 시)

#### 배치 원칙

| 원칙 | 설명 |
|------|------|
| **논리 흐름 뒤에 배치** | 데이터를 설명한 직후, 시각화로 보여준다 |
| **섹션 시작에 배치하지 않는다** | 맥락 없이 이미지부터 나오면 독자가 혼란 |
| **연속 배치 금지** | 📎 두 개가 연달아 오지 않는다 — 사이에 텍스트 최소 3줄 |
| **적정 개수** | 아티클당 3~6개 (250~300줄 기준) |

#### 배치 위치 가이드

```
§1 도입 (통념/현황)
  본문...
  📎 *[이미지: 현황 요약 차트]*     ← 도입부 데이터 뒤

§2 분해 (제1원칙)
  본문...                            ← 시각화 불필요할 수 있음

§3 Gap (데이터로 보여주기)
  본문...
  📎 *[이미지: 격차 비교 바 차트]*   ← 핵심 데이터 뒤
  본문 계속...
  📎 *[이미지: 시계열 트렌드]*       ← 추가 데이터 뒤

§4 전개 (가능성 추론)
  본문...
  📎 *[이미지: 시나리오 카드]*       ← 핵심 논점 뒤

§5 결론
  본문...                            ← 결론에는 보통 시각화 없음
```

### 2단계: 마커 매핑 테이블 작성

아웃라인(outline.md) 또는 v1 상단에 매핑 테이블을 작성한다:

```markdown
## 📎 이미지 매핑

| 📎 # | 위치 | 설명 | 소스 파일 | 상태 |
|-------|------|------|-----------|------|
| 1 | §1 뒤 | 현황 요약 차트 | visuals/01_*.png | ✅ 생성 |
| 2 | §3 중간 | 격차 비교 | visuals/02_*.png | ✅ 생성 |
| 3 | §3 뒤 | 시계열 트렌드 | visuals/03_*.png | ⏳ 미생성 |
| 4 | §4 뒤 | 시나리오 카드 | images/외부이미지.png | ⏳ 수집 필요 |
```

> **📎 번호 ≠ 파일명 번호일 수 있다.** 삽입 순서와 생성 순서가 다를 수 있으므로 반드시 매핑을 유지한다.

### 3단계: 검증 — 단일 파일 내

v1 또는 v2 작성 완료 후:

```
검증 1: 본문 📎 개수 == 매핑 테이블 행 수
검증 2: 각 📎의 설명이 매핑 테이블과 일치
검증 3: 소스 파일이 실제 존재하는지 (visuals/, images/ 확인)
검증 4: 연속 📎 없음 (사이에 텍스트 3줄 이상)
```

### 4단계: 교차 검증 — 파일 간

x_publish 생성 후, 또는 네이버 변환 후:

```
     v2 (원본)          x_publish (게시용)       naver HTML
     📎 개수      ==     📎 개수           ==    📷 플레이스홀더 수
        ↕                    ↕                         ↕
     매핑 테이블          X_IMAGE_GUIDE.md         README.md 이미지 목록
```

#### 교차 검증 체크리스트

| # | 검증 | 방법 | 불일치 시 |
|---|------|------|-----------|
| 1 | v2 📎 수 == x_publish 📎 수 | 양쪽 파일에서 `📎` 카운트 | x_publish 재생성 |
| 2 | x_publish 📎 수 == X_IMAGE_GUIDE 행 수 | GUIDE 테이블 행 카운트 | GUIDE 갱신 |
| 3 | naver HTML 📷 수 == v2 📎 수 | HTML에서 이미지 플레이스홀더 카운트 | HTML 재생성 |
| 4 | 소스 파일 전부 존재 | visuals/, images/ 파일 존재 확인 | 미생성 파일 목록 출력 |

### 5단계: 최종 리포트

검증 완료 후 결과를 출력한다:

```
📎 이미지 마커 검증 결과
─────────────────────────
아티클: {순번}_{티커}_{slug}
마커 수: {N}개

✅ v2 📎 = {N}
✅ x_publish 📎 = {N}
✅ X_IMAGE_GUIDE 행 = {N}
✅ 소스 파일 전부 존재 ({M}/{N})
⚠️ naver HTML 📷 = {K} (불일치 — 재생성 필요)

미생성 파일:
  - visuals/03_*.png (§3 시계열 트렌드)
```

## 연동

```
article-drafting (초안) ─── 📎 배치 ──→ [이 스킬] 1~2단계
generate-visuals (시각화) ─ 파일 생성 ──→ [이 스킬] 3단계
article-revision (퇴고) ── x_publish ──→ [이 스킬] 4단계
naver-packaging (변환) ─── HTML 생성 ──→ [이 스킬] 4단계
```

## 주의사항

- **📎 번호는 삽입 순서**이다 — 파일명 번호(`01_`, `02_`)와 다를 수 있다
- 퇴고에서 섹션 순서가 바뀌면 📎 매핑도 갱신해야 한다
- 외부 이미지(`images/`)는 사용자가 직접 수집한다 — Claude가 웹 검색하지 않는다
- 히어로 이미지(`00_hero_*.png`)는 📎 매핑에 포함하지 않는다 — 별도 첨부
