# 스케줄 브리핑 (schedule-briefing)

## 목적

- 현재 아티클 파이프라인 상태를 한눈에 파악하고 브리핑한다
- **마스터 테이블 vs 실제 디렉토리** 간 불일치를 감지하여 동기화 필요 여부를 판단한다
- 이번 주 게시 대상, 재고 요약, 경고, 추천 액션을 리포트한다

## 실행 조건

| 트리거 | 설명 |
|--------|------|
| "스케줄 알려줘" | 사용자가 명시적으로 요청 |
| 세션 시작 시 | 아티클 관련 작업이 예상될 때 자발적으로 실행 |
| 게시 전 점검 | 아티클 게시 직전 현황 확인 |

## 절차

### 1단계: PUBLISH_SCHEDULE.md 읽기

`docs/프로젝트/PUBLISH_SCHEDULE.md`에서 아티클 마스터 테이블을 읽는다:

| 읽는 내용 | 용도 |
|-----------|------|
| 아티클 마스터 테이블 | 전체 아티클 목록, 단계, 폴더명, 게시 예정, X/네이버 체크 |
| 주간 편성표 | 이번 주 게시 대상 파악 |

### 2단계: 실제 디렉토리 스캔

`data/articles/` 및 `data/articles/published/` 디렉토리를 스캔한다:

```
data/articles/         → 전체 아티클 폴더 목록
data/articles/published/ → 게시 대기/완료 폴더 목록
```

### 3단계: 2-way 불일치 탐지

마스터 테이블과 실제 디렉토리를 교차 대조한다:

| # | 불일치 유형 | 설명 | 심각도 |
|---|------------|------|--------|
| 1 | **유령 아티클** | 마스터에 등록돼 있으나 실제 폴더가 없음 | 🔴 |
| 2 | **미등록 폴더** | `data/articles/`에 폴더가 있으나 마스터에 없음 | ⚠️ |
| 3 | **published 미반영** | published/ 에 있는데 마스터 단계가 게시대기 이전 | ⚠️ |
| 4 | **완료 미갱신** | 마스터에서 ✅완료인데 X/네이버 체크가 `[ ]` | ⚠️ |

불일치 발견 시: 사용자에게 보고하고 `schedule-sync` 실행 여부를 확인한다.

### 4단계: 시각화 생성

```bash
cd /home/gint_pcd/projects/주식부자프로젝트
source .venv-wsl/bin/activate && python docs/프로젝트/generate_schedule_visual.py
```

생성된 PNG를 사용자에게 보여준다:

```bash
explorer.exe "$(wslpath -w docs/프로젝트/schedule_visual.png)"
```

### 5단계: 브리핑 리포트 출력

아래 형식으로 리포트를 출력한다:

```
## 📋 아티클 스케줄 브리핑 ({오늘 날짜})

### 이번 주 게시
| 요일 | 아티클 | 상태 | 남은 작업 |
|------|--------|------|-----------|
| ... | ... | ... | ... |

### 재고 요약
- 게시 가능: N편
- 퇴고 필요: N편
- 초안 필요: N편
- 기획 필요: N슬롯 (빈 슬롯)

### ⚠️ 경고 (있을 때만)
- [불일치 / 지연 / 누락 항목]

### 추천 액션
1. [가장 시급한 작업]
2. [다음 작업]
```

## 추천 액션 우선순위 로직

| 우선순위 | 조건 | 추천 |
|----------|------|------|
| 1 | 이번 주 게시 예정인데 상태가 "퇴고 필요" | "X편 퇴고 먼저" |
| 2 | 게시 가능 상태인데 아직 미게시 | "X편 게시 진행" |
| 3 | 다음 주 게시 예정인데 초안 미완 | "X편 초안 서두르기" |
| 4 | 빈 슬롯이 2주 이내 | "금요일 슬롯 기획 필요" |
| 5 | 특별한 이슈 없음 | "현재 페이스 양호" |

## 주의사항

- 이 스킬은 **읽기 전용** — 직접 파일을 수정하지 않는다 (수정은 `schedule-sync`에게 위임)
- 불일치가 발견되면 사용자에게 먼저 보고하고 sync 실행 여부를 확인한다
- 시각화 PNG 열기는 WSL 환경에서 `explorer.exe` 사용 (Windows 경로 변환 필요)
- "이번 주"는 현재 날짜 기준 해당 Week의 월~금을 의미한다
