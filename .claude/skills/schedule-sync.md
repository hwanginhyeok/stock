# 스케줄 동기화 (schedule-sync)

## 목적

- 아티클 상태가 변경될 때 **PUBLISH_SCHEDULE.md 마스터 테이블을 갱신**한다
- 갱신 후 시각화를 재생성하여 항상 최신 상태를 반영한다

## 실행 조건

| 트리거 | 설명 |
|--------|------|
| 아티클 단계 변경 | 리서치→초안, 초안→퇴고, 퇴고→게시가능, 게시→완료 등 |
| published 이동 | 아티클 폴더가 `published/`로 이동했을 때 |
| 신규 아티클 등록 | 새 아티클 폴더가 생성되었을 때 |
| briefing 불일치 발견 | `schedule-briefing` 스킬에서 불일치 감지 시 분기 |
| 게시 완료 | X 또는 네이버 게시 완료 후 마스터 테이블 체크 |

## 절차

### 1단계: 변경 유형 판단

| 유형 | 설명 | 갱신 범위 |
|------|------|-----------|
| **신규** | 새 아티클 등록 | 마스터 테이블 행 추가 + 주간편성 배정 + GRID |
| **단계변경** | 리서치→초안→퇴고→게시가능 | 마스터 테이블 `단계` 컬럼 + GRID 상태색 |
| **published 이동** | 폴더를 published/로 이동 | 마스터 테이블 `단계` → 게시대기 |
| **게시완료** | X/네이버 실제 게시 | 마스터 테이블 `X`/`네이버` 컬럼 `[x] 날짜` 체크 |
| **편성변경** | 주간 편성표 슬롯 이동/교체 | 마스터 테이블 `게시 예정` + 주간 테이블 + GRID 좌표 |

### 2단계: PUBLISH_SCHEDULE.md 마스터 테이블 갱신

갱신 대상:

| 대상 | 갱신 내용 |
|------|-----------|
| **아티클 마스터** 테이블 | `단계` 컬럼, `게시 예정` 날짜, `X`/`네이버` 체크박스 |
| `Week N` 주간 테이블 | 아티클명, 상태 (마스터와 일치) |
| `슬롯별 콘텐츠 기획 방향` | 후보 상태 업데이트 |

단계 컬럼 값 매핑:

| 단계 | 마스터 테이블 값 | 비고 |
|------|-----------------|------|
| 리서치 중 | 🔬리서치 | 초안 미작성 |
| 초안 완료 | 초안 | 퇴고 대기 |
| 퇴고 완료 | v2완료 | published 이동 대기 |
| published 이동 | 게시대기 | 게시 준비 완료 |
| 게시 완료 | ✅완료 | X/네이버 체크 포함 |
| 미착수 | 기획예정 | 폴더 미생성 |

### 3단계: generate_schedule_visual.py GRID 갱신

GRID 딕셔너리 갱신:

```python
# key: (week_idx, day_idx)  →  day_idx: 0=월, 1=화, 2=수, 3=금
# val: (series, title, series_color, status_color, status_label)

# 상태별 색상 코드 (스크립트 상단 상수)
S_READY    = "#3fb950"  # 초록 — 게시 가능
S_DRAFT    = "#d29922"  # 노랑 — 퇴고 필요
S_RESEARCH = "#58a6ff"  # 파랑 — 초안 필요
S_PLAN     = "#484f58"  # 회색 — 기획 예정
```

### 4단계: 시각화 재생성 (선택)

```bash
cd /home/gint_pcd/projects/주식부자프로젝트
source .venv-wsl/bin/activate && python docs/프로젝트/generate_schedule_visual.py
```

생성 확인: `schedule_visual.png` 파일 크기 > 10KB.

## 변경유형 × 갱신범위 매트릭스

|----------|:---:|:---:|:---:|
| 신규 등록 | ✅ | ✅ | ✅ |
| 단계 변경 | ✅ | ✅ | ✅ |
| published 이동 | ✅ | — | — |
| 게시 완료 | ✅ | ✅ | ✅ |
| 편성 변경 | ✅ | ✅ | ✅ |

## 주의사항

- **마스터 테이블이 단일 소스** — 주간 테이블과 GRID는 마스터에서 파생
- GRID의 `week_idx`는 0-based (W1=0, W2=1, ...), `day_idx`도 0-based (월=0, 화=1, 수=2, 금=3)
- PUBLISH_SCHEDULE.md의 `업데이트:` 날짜를 갱신 시점으로 변경한다
- 시각화 재생성 전에 가상환경 활성화 확인 (`source .venv-wsl/bin/activate`)
