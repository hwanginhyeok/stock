# 네이버 변환 패키징 (naver-packaging)

## 목적

- X Notes 게시 완료된 아티클을 네이버 블로그 형식으로 변환·패키징한다
- `naver/` 디렉토리에 HTML + 번호순 이미지 + README.md를 완비하여 수동 게시만 남긴다
- QA를 통해 변환 품질을 보장한다

## 실행 조건

| 트리거 | 설명 |
|--------|------|
| X Notes 게시 완료 후 | `publish-x` 스킬 실행 후 네이버 변환 시 |
| 사용자 요청 | "네이버 변환해 줘" |

**전제 조건:**
- `published/{순번}_{티커}_{slug}/` 폴더에 최종 마크다운 + 이미지가 있어야 한다
- X Notes에 먼저 게시된 상태여야 한다 (네이버는 아카이브 개념)

## 절차

### 1단계: 참조 파일 확인

| 파일 | 확인 내용 |
|------|-----------|
| `.claude/rules/naver-html.md` | HTML 형식·스타일·색상·워크플로우 사양 |
| `.claude/skills/publish-x.md` | 태그 사전 (공통 + 카테고리별) |

### 2단계: HTML 변환

```bash
cd /home/window11/stock
source .venv-wsl/bin/activate && python scripts/generate_naver_html.py <v2_마크다운_파일_경로>
```

- **입력: 반드시 `*_v2.md` (퇴고본)** — x_publish 사용 금지
- 출력: 같은 위치에 `*_naver.html` 생성

> **⚠️ 소스 파일 규칙 (필수)**
> - `v2`에는 마크다운 테이블(`| | |`)이 원본 그대로 있다 → 변환기가 zebra-stripe `<table>` HTML로 정상 변환
> - `x_publish`는 X Notes용으로 테이블을 텍스트(볼드 라벨+이모지)로 변환한 상태 → HTML에 테이블이 아예 생성되지 않음
> - v2 blockquote의 `상태: ...` 메타데이터 → **변환기가 자동 제거** (수동 작업 불필요)
> - `## 면책조항` → **변환기가 자동으로 disc 스타일 div로 변환** (회색 박스, 13px)

### 2.5단계: 이미지 마커 교차 검증

v2에서 HTML을 생성하면 x_publish에서 추가한 이미지 마커가 누락될 수 있다.

1. x_publish의 📎 마커 목록 추출
2. HTML의 📷 플레이스홀더 목록 추출
3. x_publish에만 있는 마커 → HTML에 수동 추가

### 3단계: naver/ 디렉토리 구성

```bash
# 디렉토리 생성
mkdir -p published/{순번}_{티커}_{slug}/naver/

# HTML 이동
mv published/{순번}_{티커}_{slug}/*_naver.html \
   published/{순번}_{티커}_{slug}/naver/
```

### 4단계: 이미지 복사 + 리네이밍

아티클의 이미지를 `naver/`에 번호순 한글명으로 복사한다:

```
visuals/timeline_achievements.png  →  naver/01_타임라인.png
visuals/cards_comparison.png       →  naver/02_비교카드.png
visuals/bar_growth.png             →  naver/03_성장률.png
```

**리네이밍 규칙:**
- 접두사: `01_`, `02_`, `03_` ... (삽입 순서)
- 이름: 이미지 내용을 한글로 간결하게 (예: `타임라인`, `비교카드`, `매출구조`)
- 확장자: 원본 유지 (`.png`)

### 5단계: README.md 생성

`naver/README.md`에 아래 내용을 작성한다:

```markdown
# 네이버 블로그 게시 가이드 — {아티클 제목}

## 게시 정보

| 항목 | 내용 |
|------|------|
| 아티클 | {순번} — {제목} |
| X 게시일 | {날짜} |
| HTML 파일 | {파일명} |
| 이미지 수 | {N}개 |

## 이미지 삽입 순서

| 순서 | 파일명 | 본문 위치 (📎 마커) |
|------|--------|---------------------|
| 1 | 01_타임라인.png | "📎 *[이미지: 타임라인]*" 위치 |
| 2 | 02_비교카드.png | "📎 *[이미지: 비교카드]*" 위치 |
| ... | ... | ... |

## 태그 ({N}개)

{카테고리별 공통 태그 + 아티클 고유 태그}

## 게시 체크리스트

- [ ] HTML → Chrome 열기 → Ctrl+A → Ctrl+C
- [ ] 네이버 블로그 에디터 → Ctrl+V
- [ ] 이미지 📎 플레이스홀더 위치에 이미지 삽입 (순서대로)
- [ ] 인용문(blockquote) 중앙정렬 수동 설정
- [ ] 태그 입력
- [ ] 카테고리 설정
- [ ] 미리보기 확인
- [ ] 발행
```

### 6단계: QA 점검

| # | 점검 항목 | 기준 | 확인 방법 |
|---|-----------|------|-----------|
| 1 | HTML 파일 크기 | > 1KB | `ls -la naver/*.html` |
| 2 | 이미지 수 = 📎 플레이스홀더 수 | 일치 | HTML 내 플레이스홀더 div 개수 vs 이미지 파일 수 |
| 3 | 태그 개수 | 15~30개 | README.md 태그 섹션 카운트 |
| 4 | 번호 연속성 | 01, 02, 03 ... 빈 번호 없음 | `ls naver/*.png` |
| 5 | README 완성도 | 모든 섹션 채워짐 | 직접 확인 |

### 7단계: 최종 디렉토리 구조 확인

```
published/{순번}_{티커}_{slug}/
├── article_final.md                 ← 원본 마크다운
├── *_x_publish.md                   ← X Notes용
├── visuals/                         ← 원본 시각화
│   ├── 01_*.png
│   └── ...
└── naver/                           ← 네이버 게시 패키지
    ├── *_naver.html                 ← 변환된 HTML
    ├── README.md                    ← 게시 가이드
    ├── 01_이미지명.png               ← 삽입 순서 이미지
    ├── 02_이미지명.png
    └── ...
```

### 8단계: 수동 게시 절차 안내

사용자에게 아래 수동 절차를 안내한다:

1. `naver/*.html`을 Chrome에서 연다
2. `Ctrl+A` → `Ctrl+C` (전체 선택 후 복사)
3. 네이버 블로그 에디터에서 `Ctrl+V` (붙여넣기)
4. 📎 플레이스홀더 위치에 이미지를 순서대로 삽입
5. 인용문(`<blockquote>`) → 에디터에서 중앙정렬 수동 설정
6. README.md의 태그를 입력
7. 미리보기로 최종 확인 → 발행

### 9단계: 게시 완료 후 → schedule-sync

네이버 게시 완료 시 `schedule-sync` 스킬로 게시 트래커 갱신:
- `PUBLISH_SCHEDULE.md` 게시 트래커 → 블로그 게시 `[x]` + 날짜

## 태그 작성 가이드

태그 구성 = **공통 태그** + **카테고리 태그** + **아티클 고유 태그**

| 구성 | 소스 | 개수 목안 |
|------|------|-----------|
| 공통 | 투자분석, 주식분석, 심층분석 | 3개 |
| 카테고리 | `.claude/rules/naver-html.md` 태그 사전 | 8~12개 |
| 고유 | 아티클 핵심 토픽 키워드 | 5~10개 |
| **합계** | | **15~25개** |

## 주의사항

- **SmartEditor ONE에는 HTML 탭이 없다** — 반드시 브라우저 복붙(Chrome Ctrl+A/C → 에디터 Ctrl+V) 방식 사용
- 인라인 CSS만 생존한다 — `<style>` 블록, CSS 클래스는 전부 제거됨
- `text-align` 속성은 에디터가 강제 변환 — 인용문 정렬은 수동으로
- 이미지는 에디터에서 직접 삽입해야 한다 — HTML 내 `<img>` 태그는 외부 경로 참조 불가
- 네이버는 X의 아카이브 개념 — X 기준으로 작성한 아티클을 변환하는 것이지, 네이버를 위해 원문을 바꾸지 않는다
