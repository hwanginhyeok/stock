# 아티클 마감 (article-shipping)

## 목적

- **글을 내는 것**에 집중한다 — 시작이 아니라 완료가 목표
- WIP(동시 진행) 제한으로 "12건 동시 진행, 완료 0건" 패턴을 방지한다
- 게시 후 반응을 추적하여 다음 글에 반영한다

## WIP 제한 (불변 규칙)

| 규칙 | 값 | 위반 시 |
|------|-----|---------|
| **게시 대기** 상한 | 3건 | 게시 대기 3건 이상이면 **새 리서치/초안 시작 금지**. 먼저 게시하라 |
| **진행 중(v1 이상)** 상한 | 5건 | 5건 초과 시 새 아웃라인 승인 금지 |
| **리서치만(Phase 1)** | 제한 없음 | 리서치는 자유롭게 — 단, 아웃라인 전환 시 WIP 체크 |

**WIP 체크 트리거:**
- 새 아티클 기획 시
- 아웃라인 승인 요청 시
- 세션 시작 시 (TASK.md 기반 카운트)

## 게시 cadence

| 기본 | 주 1편 |
|------|--------|
| 여유 시 | 주 2편 (최대) |
| 바쁠 때 | 격주 1편 (최소) |

**cadence 미달 시**: 2주 연속 게시 0건이면 가장 완성도 높은 게시 대기 글을 **강제 게시** 대상으로 제안.

## 마감 타임박스 (v1 이후)

v1 초안 완료 후, 게시까지의 각 단계에 시간 제한을 둔다:

| 단계 | 담당 | 타임박스 | 초과 시 |
|------|------|---------|---------|
| 퇴고 (v1→v2) | AI 실행 → 사용자 검토 | 세션 1회 | 2회 퇴고 후 강제 마감 (v3이 최종) |
| 팩트체크 | AI 실행 → 사용자 확인 | 퇴고와 동시 | — |
| 시각화 | AI 생성 → 사용자 QA | 세션 1회 | 시각화 없이 게시 가능 (텍스트만) |
| x_publish 변환 | AI 자동 | 즉시 | — |
| 네이버 변환 | AI 자동 | 즉시 | 생략 가능 — X 먼저 게시 |

**"완벽보다 게시"**: 무한 수정 루프 방지. v1→v2→v3(최대). v3에서 강제 마감.

## 게시 순서 원칙

1. **준비도 순**: 가장 적은 작업으로 게시 가능한 글부터
2. **Tesla 우선**: 같은 준비도면 Tesla 관련 글이 먼저 (채널 정체성)
3. **시리즈 순서**: 같은 시리즈 내에서는 번호 순

## 게시 후 학습 (Post-Publish Review)

게시 후 1주일 내에 아래를 기록한다:

### 기록 위치

`docs/프로젝트/task/4-x.md` → `## 게시 후 리뷰` 섹션

### 기록 항목

```markdown
### 게시 후 리뷰 — {날짜}

**플랫폼**: X Notes / 네이버
**게시일**: YYYY-MM-DD

**반응 (정량):**
- 조회수:
- RT/공유:
- 좋아요:
- 댓글 수:

**반응 (정성):**
- 가장 많이 인용된 구절/데이터:
- 댓글 중 인사이트:
- 비판/반론:

**자가 평가:**
- 이 글에서 가장 잘한 것:
- 다음에 개선할 것:
- 반응이 예상과 달랐다면 왜:
```

### 학습 → 시스템 반영

게시 후 리뷰에서 패턴이 나오면:
1. "이런 구조의 글이 반응이 좋다" → `article-writing.md`에 반영
2. "이 톤이 더 먹힌다" → `writing-voice.md`에 반영
3. "이 주제가 관심을 끌었다" → 콘텐츠 우선순위에 반영

## 세션 시작 시 마감 체크

매 세션 시작 시:

```
1. 게시 대기 건수 확인 (≥3이면 새 작업 차단)
2. 마지막 게시일 확인 (2주 이상이면 강제 게시 제안)
3. 가장 완성도 높은 대기 글 식별
4. "오늘 이 글을 마감합시다" 제안
```

## 연동

```
article-revision (퇴고 완료)
    → [이 스킬] 게시 타임박스 체크
    → publish-x (X 게시)
    → naver-packaging (네이버 변환)
    → [이 스킬] 게시 후 학습
    → feedback-tracking (패턴 → 시스템 반영)
```

## 주의사항

- WIP 제한은 **불변 규칙** — 사용자가 명시적으로 해제하지 않는 한 적용
- "나중에 고칠게"는 마감 회피의 신호 — 2회 퇴고 후 강제 마감
- 시각화 없이 게시하는 것은 허용된다 — 텍스트 품질이 핵심
- 네이버 변환은 항상 선택 — X 게시가 필수, 네이버는 여건 될 때
