# 네이버 블로그 HTML 가이드

> 마크다운 아티클 → 네이버 블로그 HTML 변환 시 따르는 형식·스타일 기준.
> 변환기: `scripts/generate_naver_html.py`

---

## 네이버 SmartEditor ONE 제약 사항

| 제약 | 설명 |
|------|------|
| **인라인 CSS only** | `<style>` 블록, `<link>` 태그, CSS 클래스 → 전부 제거됨. `style="..."` 속성만 생존 |
| **HTML 편집 탭 없음** | 2022년 11월 이후 SmartEditor ONE에서 HTML 탭 완전 제거 |
| **`<table>` 제한** | 에디터 내 테이블 위젯과 충돌 가능. 브라우저 복붙 시 대부분 유지됨 |
| **`<script>` 금지** | JavaScript 완전 차단 |
| **iframe 금지** | 외부 삽입 불가 |
| **스티커/지도** | 네이버 내부 컴포넌트 — HTML/API로 삽입 불가, 에디터에서만 가능 |
| **`text-align` 무시** | 붙여넣기 시 정렬을 자체 클래스로 강제 변환 → 인용문 중앙정렬은 에디터에서 수동 설정 |

## HTML 삽입 방법 (3가지)

| 방법 | 설명 | 추천 상황 |
|------|------|-----------|
| **브라우저 복붙** | HTML → Chrome 열기 → `Ctrl+A` → `Ctrl+C` → 에디터 `Ctrl+V` | **기본 방법** (가장 안정적) |
| **DevTools 주입** | 에디터에서 `F12` → Inspector → `<div class="se-module-text">` 찾기 → Edit as HTML | 정밀 제어 필요 시 |
| **MetaWeblog API** | XML-RPC API로 HTML 직접 전송 (n8n 자동화 가능) | 자동화 파이프라인 |

## 벤치마크: 투자 분석 블로그 스타일

네이버 인기 투자 블로그("부자가 되어봅시다" 등)에서 추출한 포맷 기준:

- **폰트**: 나눔고딕 (시스템 폰트, se-ff-system)
- **본문 크기**: 15px (se-fs-fs15)
- **소제목 크기**: 19px bold (se-fs-fs19)
- **정렬**: 좌측 (se-ta-left)
- **구분선**: 가는 실선 (line1)
- **키워드 강조**: `<strong>` bold
- **장식**: 스티커/이모지 없음, 데이터 중심

## 스타일 사양 (인라인 CSS)

### 레이아웃

```css
/* 전체 래퍼 */
font-size: 16px;
line-height: 2.0;
color: #333;
word-break: keep-all;
text-align: left;
font-family: '나눔고딕', 'Nanum Gothic', sans-serif;
max-width: 720px;
margin: 0 auto;
padding: 0 16px;
```

### 헤딩 (4단계 위계)

| 태그 | 크기 | 굵기 | 색상 | 여백 (상/하) | 비고 |
|------|------|------|------|-------------|------|
| `h1` | 26px | 700 | #222 | 48px / 20px | 하단 보더 2px solid #333 |
| `h2` | 21px | 700 | #222 | 40px / 16px | — |
| `h3` | 19px | 700 | #333 | 32px / 12px | — |
| `h4` | 16px | 700 | #444 | 24px / 10px | — |

- 모든 헤딩: `line-height: 1.6`
- `h1`만 하단 보더라인 있음 (글 제목 역할)

### 본문

| 요소 | 크기 | 색상 | 여백 | 비고 |
|------|------|------|------|------|
| `<p>` | 16px (상속) | #333 (상속) | 하단 20px | `line-height: 2.0` |
| `<strong>` | — | #222 | — | `font-weight: 700` |
| `<em>` | — | #555 | — | `font-style: italic` |
| `<del>` | — | — | — | 취소선 (스타일 없음) |

### 인용문 (blockquote)

```css
margin: 24px 0;
padding: 16px 20px;
background: #f7f8fa;
border-left: 3px solid #333;
color: #444;
font-size: 20px;
line-height: 2.0;
```

### 구분선 (hr)

```css
border: none;
border-top: 1px solid #ddd;
margin: 36px 0;
```

### 목록

| 요소 | 스타일 |
|------|--------|
| `<ul>` / `<ol>` | `margin: 0 0 20px; padding-left: 24px` |
| `<li>` | `margin-bottom: 10px; line-height: 2.0` |

### 테이블

| 요소 | 크기 | 배경 | 테두리 | 패딩 |
|------|------|------|--------|------|
| `<table>` | 15px | — | collapse | — |
| `<th>` | — | #f7f8fa | 1px solid #e0e0e0 | 10px 14px |
| `<td>` 홀수행 | — | 없음 | 1px solid #e0e0e0 | 9px 14px |
| `<td>` 짝수행 | — | #fafbfc | 1px solid #e0e0e0 | 9px 14px |

- 짝수행 배경색(zebra stripe)으로 가독성 확보
- `<th>`: `font-weight: 700; color: #222; text-align: left`
- `<td>`: `color: #444`

### 코드

| 요소 | 크기 | 배경 | 비고 |
|------|------|------|------|
| 인라인 `<code>` | 14px | #f2f3f5 | `color: #c7254e; border-radius: 3px; padding: 2px 6px` |
| 블록 `<pre>` | 14px | #f7f8fa | `border: 1px solid #e0e0e0; border-radius: 6px; padding: 16px 20px` |

### 이미지 플레이스홀더

```css
margin: 24px 0;
padding: 20px;
text-align: center;
background: #f0f0f0;
border: 1px dashed #ccc;
border-radius: 6px;
color: #999;
font-size: 13px;
```

- 마크다운에서 `📎 *[이미지: 설명]*` 또는 `> 📎 설명` 패턴 감지
- HTML 변환 시 회색 박스로 렌더링 → 에디터에서 실제 이미지로 교체

### 면책조항

```css
margin-top: 40px;
padding: 14px 18px;
background: #f9f9f9;
border: 1px solid #eee;
border-radius: 6px;
font-size: 13px;
color: #888;
line-height: 1.7;
```

## 마크다운 → HTML 매핑

| 마크다운 | HTML 출력 | 비고 |
|----------|-----------|------|
| `# 제목` | `<h1>` | 글 대제목 |
| `## 제목` | `<h2>` | 섹션 제목 |
| `### 제목` | `<h3>` | 소제목 |
| `#### 제목` | `<h4>` | 세부 제목 |
| `**굵게**` | `<strong>` | — |
| `*기울임*` | `<em>` | — |
| `` `코드` `` | `<code>` | — |
| `~~취소~~` | `<del>` | — |
| `---` / `***` | `<hr>` | — |
| `> 인용` | `<blockquote>` | 연속 줄 `<br>` 연결 |
| `- 항목` | `<ul><li>` | — |
| `1. 항목` | `<ol><li>` | — |
| `\| 테이블 \|` | `<table>` | 구분선 행 자동 제거 |
| `` ``` 코드블록 ``` `` | `<pre><code>` | — |
| `📎 *[이미지]*` | `<div>` 플레이스홀더 | 에디터에서 교체 |
| 연속 빈 줄 (2줄+) | `<p>&nbsp;</p>` × (N-1) | 소스에 명시적 간격이 있을 때 추가 여백 |

### 자동 의미 간격 (2026-03-09 추가)

네이버 에디터는 CSS `margin`을 일부 무시하므로 `<p>&nbsp;</p>` **물리적 빈 줄**이 가장 신뢰할 수 있는 간격 수단이다.
변환기가 아래 위치에 **자동으로** 스페이서를 삽입한다 (소스에 빈 줄이 없어도 동작).

| 위치 | 삽입 | 효과 |
|------|------|------|
| `<h1>` / `<h2>` / `<h3>` 앞 | `<p>&nbsp;</p>` 1개 | 섹션 전환 시 시각적 구분 |
| `<hr>` 앞 | `<p>&nbsp;</p>` 1개 | 구분선 전 여백 확보 |
| `<blockquote>` 앞뒤 | 각 `<p>&nbsp;</p>` 1개 | 인용문 강조 |
| `<table>` 뒤 | `<p>&nbsp;</p>` 1개 | 데이터→본문 전환 |

**중복 방지**: 직전 출력이 이미 `&nbsp;` 스페이서인 경우 추가 삽입 안 함.
**소스 빈 줄과의 관계**: 소스에 연속 빈 줄(2줄+)이 있으면 자동 간격 **위에** 추가 스페이서가 쌓인다.

## 변환 워크플로우

> **⚠️ 소스 파일 규칙 (필수)**
> 네이버 HTML은 반드시 **v2 (퇴고본)**에서 생성한다. x_publish 사용 금지.
> - v2: 마크다운 테이블 원본 → `<table>` zebra-stripe HTML 정상 변환
> - x_publish: 테이블이 이미 텍스트로 변환된 상태 → HTML에 테이블 없음 (가독성 저하)
> - v2 blockquote의 `상태: ...` 메타데이터 → **변환기가 자동 제거** (수동 작업 불필요)
> - `## 면책조항` → **변환기가 자동으로 disc 스타일 div로 변환** (회색 박스, 13px)

```
1. 마크다운 아티클 작성 (published/{순번}_{티커}_{slug}/)
2. 변환 실행 (v2 파일 사용):
   python scripts/generate_naver_html.py <*_v2.md 파일>
3. HTML QA 체크리스트 수행 (아래 참조)
4. HTML을 naver/ 하위 디렉토리로 이동
5. 이미지 파일을 naver/에 번호순으로 복사
6. README.md 작성 (이미지 매핑 + 태그 + 체크리스트)
```

### HTML 생성 후 QA 체크리스트 (필수)

| # | 항목 | 기준 | 확인 방법 |
|---|------|------|-----------|
| 1 | 면책조항 스타일 | 회색 박스(#f9f9f9) + 작은 글씨(13px) | HTML에서 `disc` 스타일 div 확인 |
| 2 | 이미지 수 일치 | HTML 📷 수 = x_publish 📎 수 | grep 카운트 교차 비교 |
| 3 | 상태 메타데이터 | blockquote에 "상태:" 없음 | grep "상태" 확인 |
| 4 | 테이블 렌더링 | `<table>` 태그 존재 + zebra stripe | grep "<table" 카운트 |
| 5 | 파일 크기 | > 1KB | ls -la 확인 |

### 디렉토리 구조 (게시 준비 완료 상태)

```
published/{순번}_{티커}_{slug}/
├── article_final.md                  ← 원본 마크다운
├── 01_이미지명.png                    ← 시각화 이미지들
├── ...
└── naver/                            ← 네이버 블로그 게시 패키지
    ├── article_final_naver.html      ← 변환된 HTML
    ├── README.md                     ← 게시 가이드 (이미지 순서 + 태그 + 체크리스트)
    ├── 01_이미지명.png                ← 삽입 순서대로 번호 매긴 이미지
    ├── 02_이미지명.png
    └── ...
```
