## 설계 명세

### EntityType 재구성 (기존 8종 → 9종)

기존 유지: company, person, asset, institution, sector, country, proxy, commodity
**추가**: `capability` (비물질적 역량 — FSD, Dojo, Tesla Vision, 4680 Cell)

### Tesla 특화 속성 스키마 (ontology_config.yaml 확장)

```yaml
tesla_product:
  essential:
    - category           # vehicle/energy/charging
    - first_principle_bet  # 어떤 제1원칙에 기반하는가
    - role_in_master_plan  # Master Plan 3에서의 역할
  propria:
    - unit_volume
    - asp                # 평균 판매가
    - margin
    - production_ramp_rate

tesla_capability:
  essential:
    - moat_type          # intangible_assets/network_effects/cost_advantage
    - inputs_required    # 필요 입력 (데이터, 컴퓨트, 인재)
    - output_function    # 산출 기능
  propria:
    - accuracy_metric
    - users
    - interventions_per_mile

tesla_factory:
  essential:
    - production_mandate  # 무엇을 만드는 공장인가
    - vertical_scope      # 어디까지 내재화했나
    - product_mix         # 생산 제품 조합
  propria:
    - capacity
    - utilization
    - opex
    - workforce

tesla_initiative:
  essential:
    - strategic_goal      # 전략적 목표
    - timeline_horizon    # 단기/중기/장기
    - essence_component   # 4대 본질 중 관련 항목
    - capital_required_range  # 필요 자본 규모
  propria:
    - progress_pct
    - milestone_count
    - burn_rate
```

### 이슈 번호 체계

`TSLA-{카테고리}-###`
- E: essence (본질 변화)
- P: product
- C: capability
- F: factory
- I: initiative
- R: regulatory
- M: Musk 발언

필수 필드: `essence_component` — 4대 본질 중 영향 주는 것 (없으면 noise)

### Competency Questions (검증용)

1. Tesla essence 4개 중 현재 약화된 것은?
2. Master Plan Part 3 initiative별 진행도?
3. 오늘 실적이 어떤 essence에 영향?
4. Musk 트윗이 noise인가 essence 변화인가?
5. 경쟁자 움직임이 Tesla moat 위협?
6. FSD 지연이 Robotaxi에 미치는 영향? (capability → initiative 체인)
