# 1-23 생태계 기업 모니터링

> 상태: 예정 (보류)
> 등록일: 2026-03-01

## 작업 요약

크립토/블록체인 생태계에 실질적 영향을 주는 핵심 기업들의 가격 추이와 주요 뉴스를 추적.

## 배경

사용자 요청: "단순 트레저리 기업 말고, 뭔가 생태계에 영향을 주고자 하는 키업들만"

- 단순 BTC 보유사(MicroStrategy류) 제외
- 인프라·거래소·결제·규제 영향력이 있는 기업 포함

## 확정 모니터링 기업 (7개)

| 티커 | 기업 | 생태계 역할 |
|------|------|------------|
| `COIN` | Coinbase | 미국 최대 거래소, 기관 커스터디, 규제 대응 최전선 |
| `HOOD` | Robinhood | 리테일 크립토 진입점, 미국 개인 투자자 심리 바로미터 |
| `BMNR` | Bit Brother (BMNR) | 블록체인 인프라 (채굴 제외 기준) |
| `SQ` | Block (Square) | BTC 결제·보유 + Cash App 크립토 채널 |
| `BLK` | BlackRock | IBIT ETF 운용, 기관 자금 크립토 유입 관문 |
| `MSTR` | MicroStrategy | BTC 대규모 트레저리 + 사실상 레버리지 BTC 프록시 |
| `CRCL` | Circle | USDC 발행사, 스테이블코인 생태계의 핵심 인프라 |

> **제외 결정**: Bitmain (비상장, 중국 채굴 기업) — 생태계 영향 있으나 데이터 접근성 한계

## 설계 방향

### 가격 추이
- yfinance로 일별 OHLCV 수집 (기존 인프라 재사용)
- 7개 티커 → 주간/월간 수익률 계산
- 미니 스파크라인 or 수익률 테이블 형태로 이메일에 표시

### 뉴스 추적
- Circle (CRCL): 비상장 → RSS/뉴스 API 필요
- 나머지: yfinance `.news` or 기존 뉴스 수집기 연동 가능

### 이메일 섹션 배치
- Section 5 (크립토) 하단에 별도 카드 그룹으로 추가
- 또는 별도 Section 6로 분리 (분량에 따라 결정)

## 보류 이유

현재 Section 5가 충분히 잘 구성되어 있고,
생태계 기업 모니터링은 별도 섹션 필요성이 낮아 우선순위 P2 유지.

착수 조건:
- 스테이블코인 발행량 추적 (별도 TASK) 구현 완료 후 함께 묶을 가능성
- 또는 사용자가 명시적 착수 요청 시
