### 동기화 블로커

**상태**: HIH_2 git index에 `HIH_Claude`가 gitlink(`160000 commit 7ffe6a51...`)로 등록돼 있지만
`.gitmodules` 파일이 없어서 원격 URL 미정의 → 집 PC(window11)에서 `git submodule update` 불가.

**필요한 회사 액션** (gint_pcd 노트북):
1. `HIH_Claude` 디렉토리가 별도 git repo인지 확인 (`git remote -v`)
2. 있으면 원격 URL 확인 후 `.gitmodules` 재생성:
   ```
   [submodule "HIH_Claude"]
       path = HIH_Claude
       url = <원격 URL>
   ```
3. `.gitmodules` 커밋 & push
4. 집 PC에서 `git pull && git submodule update --init --recursive`

## Part 3: 매핑 설계 (⏳ Part 2 완료 후)

### 잠정 설계안 (Part 2 이전 초안)

이슈 DB 스타일을 stock에 맞게 재해석:

**[A] 이슈 번호 체계**
- HIH_2: `#187` 단일 시퀀스
- stock 제안: 카테고리 prefix — `GEO-001`(이란 전쟁), `TSLA-001`(Robotaxi), `MKT-001`(유동성 전환)

**[B] 이슈 테이블 → OntologyEvent 확장**

```python
class OntologyIssue(BaseEntity):      # NEW
    issue_id: str                      # GEO-001
    title: str
    category: Literal["GEO", "TSLA", "MKT", "US", "KR"]
    status: IssueStatus                # open/developing/resolved/stale
    severity: Severity                 # 기존 재사용
    owner: str = ""                    # 담당 에이전트/사람
    created_at: datetime
    last_updated: datetime
    deadline: datetime | None
    blocker: str = ""                  # 블로커 사유
    milestones: list[Milestone]        # 마일스톤 타임라인
    related_events: list[str]          # OntologyEvent IDs
    related_entities: list[str]        # OntologyEntity IDs
    notion_page_id: str = ""           # Notion 쓰기/동기화용
```

**[C] 마일스톤**

```python
class Milestone(BaseEntity):          # NEW
    issue_id: str
    title: str
    target_date: datetime
    actual_date: datetime | None
    status: Literal["planned", "in_progress", "done", "delayed"]
    evidence_urls: list[str]
```

**[D] Notion 연동**
- Notion DB 3개 생성: "이슈", "마일스톤", "엔티티 리뷰"
- `stock-notion-sync` 스킬 (hih-notion-sync 복제 + 수정)
- 동기화 방향: 초기에는 로컬 → Notion 단방향, 안정화 후 양방향

**[E] 테슬라 특화 엔티티 하위 분류** (1-44에서 확정)

```yaml
# config/ontology_config.yaml 확장안
tesla_entities:
  product:                             # 새 EntityType 또는 company의 하위
    essential: [category, launch_year, technology_stack]
    propria: [unit_volume, asp, margin]
    examples: [Model Y, Cybertruck, Optimus, Megapack, FSD]
  factory:                             # 새 EntityType
    essential: [capacity, location, product_mix]
    propria: [utilization, opex]
    examples: [Fremont, Shanghai, Berlin, Austin, TeraFab]
  initiative:                          # 장기 프로젝트
    essential: [timeline, strategic_goal, capital_required]
    propria: [progress_pct, burn_rate]
    examples: [Robotaxi, Optimus Gen2, Dojo]
```
