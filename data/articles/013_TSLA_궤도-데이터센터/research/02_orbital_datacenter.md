# 궤도 데이터센터 — 우주에서 연산하는 시대

> 수집일: 2026-03-02
> 용도: 아티클 002 (우주편) 리서치 노트 2/3
> 범위: LEO 데이터센터의 물리적 이점, 현재 진행 중인 프로젝트, 실현 조건

---

## 1. 왜 우주에서 데이터센터를 짓는가

### 지구 데이터센터의 한계

| 문제 | 현황 |
|------|------|
| 전력 소비 | 글로벌 전력의 **1~1.5%**, AI로 급증 중 |
| 냉각 비용 | 총 에너지의 **30~40%**가 냉각용 |
| 물 소비 | MS 단일 DC: **하루 120만 갤런** (올림픽 수영장 ~2개) |
| 부지 확보 | 미국 주요 지역 전력 부족, 인허가 지연 |
| AI 수요 증가 | GPU 연산 수요 연 **4~5배** 성장 |

**핵심 병목**: 전력과 냉각. AI 시대에 둘 다 기하급수적으로 증가한다.

### 우주가 제공하는 해결책

| 이점 | 지구 | LEO |
|------|------|-----|
| **냉각** | 총 에너지 30~40% 소비 | **0%** — 우주 복사 방열 (4K 배경) |
| **태양광** | 날씨·밤·대기 흡수 | **24/7 무제한** (ISS 기준 ~1,361 W/m²) |
| **부지** | 토지 비용, 인허가, NIMBY | **무한** |
| **통신** | 해저 케이블 → 광속의 2/3 | **위성 간 레이저 링크 → 광속의 ~99%** |

### 냉각 비용 0% — 이것만으로도 혁명적

지구에서 1MW GPU 클러스터를 운영하면 ~300~400kW가 냉각에 쓰인다.
LEO에서는 복사 패널만으로 냉각이 가능하다. 대류도, 전도도 필요 없다.
우주의 배경 온도는 **2.7K (-270°C)** — 지구 어디에도 없는 무료 열 싱크.

## 2. 핵심 기업 & 프로젝트

### Lumen Orbit

- **설립**: 2024년
- **목표**: AI 추론용 GPU를 LEO 위성에 탑재
- **모델**: 지상 학습 → 우주에서 추론 (latency-sensitive 작업)
- **자금**: $1.1M 시드 라운드 (2024)
- **기술**: 소형 위성 + GPU 모듈 + 태양광 패널 + 복사 냉각
- **상태**: 초기 단계, 프로토타입 개발 중

### Starcloud (Axiom Space 파트너십)

- **개념**: ISS 후속 상업 우주정거장에 GPU 랙 탑재
- **규모**: 초기 NVIDIA H100 수십~수백 기 계획
- **이점**: 기존 우주정거장 인프라 활용 (전력, 열관리, 통신)

### xAI + SpaceX 합병 시사점

- **xAI**: Grok AI 운영, 멤피스에 세계 최대급 GPU 클러스터 구축 중
- **SpaceX**: Starlink 위성 6,000+기, 100만기 FCC 신청
- **합병 (2025)**: 연산(xAI) + 수송(SpaceX) + 통신(Starlink) 수직통합
- **시사점**: 위성 간 레이저 링크 위에 분산 컴퓨팅 → "궤도 클라우드"

### 기타 참여자

| 기업 | 접근 방식 |
|------|-----------|
| Microsoft Azure Space | 지상-우주 하이브리드 클라우드 |
| AWS Ground Station | 위성 데이터 수신/처리 |
| Rivada Space Networks | 레이저 메쉬 네트워크 (600기 계획) |
| Aethero | 우주용 엣지 컴퓨팅 하드웨어 |

## 3. TRL (기술 성숙도) 평가

| 기술 요소 | TRL | 근거 |
|-----------|-----|------|
| 우주용 GPU 보드 | **5~6** | Aethero 프로토타입, 방사선 차폐 설계 존재 |
| 복사 냉각 시스템 | **7~8** | ISS에서 수십 년간 운용 (열 복사 패널) |
| 태양광 전력 | **9** | ISS, 위성에서 완전 입증 |
| 위성 간 레이저 통신 | **7~8** | Starlink V2에 탑재, 운용 중 |
| 대규모 구조물 조립 (궤도) | **6~7** | ISS 조립 경험, Axiom 모듈 |
| 장기 무인 운용 | **7~8** | GPS, 통신위성 수십 년 운용 |

**종합**: 핵심 기술 대부분 TRL 7 이상. **새로운 물리학이 필요 없다.**

## 4. 경제성 분석

### 비용 구조 비교

| 항목 | 지구 DC (1MW) | 궤도 DC (1MW) |
|------|--------------|--------------|
| 건설 | $10~15M | $50~200M (발사 포함, 현재) |
| 냉각 전력 (연간) | $2~3M | **$0** |
| 토지/임대 | $1~2M/년 | $0 |
| 통신 | 해저 케이블 | Starlink 메쉬 (포함) |
| 유지보수 | 인력 + 부품 | 로봇 + 교체 위성 |

**손익분기 조건**: 발사 비용이 $500/kg 이하로 내려가면, 냉각비 절감만으로 5년 내 투자 회수 가능.

### Starship이 바꾸는 방정식

| Starship $/kg (LEO) | 1MW DC 발사비 (~50톤) | 5년 냉각비 절감 | 순이익 |
|---------------------|----------------------|----------------|--------|
| $2,700 (현재 F9) | $135M | $12.5M | ❌ 적자 |
| $500 | $25M | $12.5M | ⚠️ 거의 균형 |
| **$100** | **$5M** | **$12.5M** | **✅ 흑자** |
| $20 | $1M | $12.5M | ✅ 대폭 흑자 |

**$100/kg이면 경제성이 성립한다.** 이것이 Starship의 초기 목표치와 일치한다.

## 5. Starlink과의 시너지 — 궤도 인터넷 백본

### 현재 Starlink 규모

- 운용 위성: **6,000+기**
- FCC 승인 신청: **42,000기** (1세대 + 2세대)
- 장기 신청: **100만 기** (V3)

### 위성 간 레이저 링크

- Starlink V2 Mini부터 탑재
- **광속의 ~99%** 전송 (광섬유는 ~66%)
- 장거리 통신에서 해저 케이블보다 **40~50% 빠른 레이턴시**
- 뉴욕-런던: 광섬유 ~65ms → Starlink 레이저 ~40ms

### 궤도 DC + Starlink = 분산 컴퓨팅 메쉬

GPU가 궤도에 있고, 통신이 레이저 링크로 연결되면:
- 사용자 → 가장 가까운 Starlink 위성 → 궤도 DC → 결과 반환
- 지상 광섬유 경유 없이 **엔드-투-엔드 우주 경로**
- AI 추론처럼 latency-sensitive하지만 throughput이 크지 않은 작업에 최적

## 6. 과제 & 제약

### 방사선

- LEO: 반 앨런 벨트 아래, 상대적으로 낮은 방사선
- 그러나 GPU/DRAM은 방사선에 취약 (비트 플립, 래치업)
- **해결**: 방사선 차폐 + ECC 메모리 + TMR(삼중 모듈 중복)
- ISS 컴퓨터는 수십 년간 방사선 환경에서 운용됨 — 해결 가능한 공학 문제

### 대역폭

- 현재 Starlink: 사용자당 ~100~300 Mbps
- 대규모 데이터 전송 (학습용 데이터셋 업로드)에는 부족
- **대안**: 학습은 지상, 추론만 궤도 (데이터가 적음)
- 향후 V3 Starlink: 대역폭 대폭 증가 예상

### 유지보수

- 고장 시 현장 수리 불가 (초기에는)
- **모듈식 설계**: 고장 모듈 교체 → 새 위성 발사
- Starship 비용이 충분히 내려가면, "수리"보다 "교체"가 경제적

## 7. 아티클에서의 위치

### 서사 흐름

```
"AI의 전력 수요가 기하급수적으로 증가한다.
 지구의 전력과 냉각은 한계가 있다.

 그런데 우주에는:
 - 냉각이 공짜다 (2.7K 배경)
 - 태양광이 24/7이다
 - 부지가 무한하다

 데이터센터가 우주로 가는 건 '공상과학'이 아니라
 물리학이 가리키는 방향이다.

 그리고 이것을 가능하게 만드는 건? Starship."
```

### 핵심 수치 (아티클용)

- 지구 DC 냉각 비용: 총 에너지의 30~40%
- 우주 DC 냉각 비용: **0%**
- Starlink 레이저 통신: 광섬유 대비 **40~50% 낮은 레이턴시**
- 경제성 성립 조건: **$100/kg** (Starship 초기 목표와 일치)
- xAI + SpaceX 합병: 연산 + 수송 + 통신 수직통합
