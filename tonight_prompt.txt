# 야간 작업 지시서: 실시간 뉴스 수집 + 타임라인 시스템 구축

> TASK 번호: 1-25
> 프로젝트: ~/stock (주식부자프로젝트)
> 우선순위: P2
> 예상 산출물: 7개 파일 신규 + 3개 파일 수정

---

## 목표

실시간 뉴스 수집 스케줄러 + 파이프라인 강화 + 타임라인 서비스를 구현한다.
현재는 모닝 워크플로우에서 하루 1회 배치 수집만 하고 있다.
이를 장중 30분 / 장외 2시간 간격의 자동 수집 데몬으로 전환하고,
수집 → 중복제거 → 티커 추출 → 감성분석 → DB 저장 파이프라인을 완성한다.

---

## 현재 상태 (코드 읽기 완료)

### 있는 것
- `src/collectors/news/rss_collector.py` — RSS 수집기 (7개 소스: KR 4, US 3)
- `src/collectors/news/dedup.py` — TitleDeduplicator (difflib, 0.85 threshold)
- `src/storage/news_repository.py` — CRUD + get_latest/get_by_market/get_by_date_range/get_recent_titles
- `src/analyzers/sentiment.py` — SentimentAnalyzer (Claude Haiku, -1.0 ~ +1.0)
- `src/core/database.py` — NewsItemDB ORM (id, title, content, summary, source, url, category, importance, sentiment_score, related_tickers, related_sectors, market, published_at, created_at)
- `src/core/models.py` — NewsItem Pydantic 모델
- `config/settings.yaml` → schedule.news_collection: {market_hours_interval_min: 30, off_hours_interval_min: 120}
- `config/news_sources.yaml` — 7개 RSS 소스 + collection 설정

### 없는 것 (구현 대상)
- 스케줄러 (APScheduler) — 설정만 있고 연결 안 됨
- 티커 자동 추출 — related_tickers 항상 빈 배열
- 수집-감성분석 파이프라인 통합 — 감성분석기가 수집 후 자동 실행 안 됨
- 타임라인 서비스 — 시간순 뉴스 피드, 티커별/시장별 필터
- 브레이킹 뉴스 감지 — 동일 이벤트 다수 소스 보도 시 중요도 자동 상향
- CLI 스크립트 — 데몬 실행, 타임라인 조회

## 구현 계획 (4 Phase, 순서대로 진행)

### Phase 1: 티커 추출기 (신규)

**파일**: `src/collectors/news/ticker_extractor.py`

**요구사항**:
- `config/market_config.yaml`에서 watchlist 로딩 (KR: 005930=삼성전자 등, US: AAPL=Apple 등)
- 뉴스 title + summary에서 종목명/티커 매칭
- KR: 한글 종목명 매칭 ("삼성전자" → "005930")
- US: 티커 심볼 매칭 ("TSLA", "Tesla" → "TSLA")
- 대소문자 무시, 부분 매칭 방지 (단어 경계 \b 사용)
- `extract_tickers(news_item: NewsItem) -> list[str]` 반환

**참고 코드 패턴**:
```python

        # config.market.us.watchlist → list[WatchlistItem(ticker, name)]
```

### Phase 2: 뉴스 파이프라인 (신규)

**파일**: `src/collectors/news/news_pipeline.py`

**요구사항**:
- 전체 파이프라인 통합: 수집 → 중복제거 → 티커 추출 → 감성분석 → DB 저장
- 감성분석은 선택적 (`analyze_sentiment: bool = True`)
  - API 비용 고려: 기본 OFF, CLI에서 `--with-sentiment` 옵션으로 켤 수 있게
- 수집 결과에 티커 자동 매핑
- 브레이킹 뉴스 감지: 최근 1시간 내 동일 키워드 3개+ 소스에서 보도 → importance=HIGH
- 파이프라인 결과 요약 로그 (수집 N건, 중복 M건 제거, 신규 K건 저장)

**핵심 메서드**:
```python
class NewsPipeline:
    def run(self, market: Market | None = None, analyze_sentiment: bool = False) -> PipelineResult
    def _detect_breaking(self, items: list[NewsItem]) -> list[NewsItem]
    def _extract_and_assign_tickers(self, items: list[NewsItem]) -> list[NewsItem]
```

**PipelineResult** (dataclass):
```python
@dataclass
class PipelineResult:
    collected: int
    duplicates_removed: int
    stored: int
    breaking_count: int
    sentiment_analyzed: int
    elapsed_sec: float
```

### Phase 3: 뉴스 스케줄러 데몬 (신규)

**파일**: `src/workflows/news_scheduler.py`

**요구사항**:
- APScheduler 사용 (BlockingScheduler)
- 의존성 추가: `pip install apscheduler` 필요
- 장중/장외 판단: KST 기준
  - KR 장중: 월~금 09:00~15:30 KST → 30분 간격
  - US 장중: 월~금 23:30~06:00 KST (다음날) → 30분 간격 (써머타임 고려)
  - 장외: 120분 간격
- 시작 시 즉시 1회 수집 실행
- graceful shutdown (SIGINT/SIGTERM)
- 실행 로그: 각 수집 사이클 결과 출력

**시장 시간 판단 로직**:
```python
from zoneinfo import ZoneInfo

KST = ZoneInfo("Asia/Seoul")
EST = ZoneInfo("America/New_York")

def is_market_hours() -> bool:
    """KR 또는 US 중 하나라도 장중이면 True."""
    now_kst = datetime.now(KST)
    now_est = datetime.now(EST)
    weekday = now_kst.weekday()  # 0=월 ... 4=금

    if weekday > 4:  # 주말
        return False

    # KR: 09:00~15:30 KST
    kr_open = now_kst.hour >= 9 and (now_kst.hour < 15 or (now_kst.hour == 15 and now_kst.minute <= 30))

    # US: 09:30~16:00 EST (DST 자동 반영 — zoneinfo가 처리)
    us_open = (now_est.hour >= 9 and now_est.minute >= 30 if now_est.hour == 9 else now_est.hour >= 10) and now_est.hour < 16

    return kr_open or us_open
```

## 주의사항

- RSS 소스 중 일부(네이버 금융뉴스, 연합뉴스)는 RSS가 아닌 HTML 페이지일 수 있음 — 수집 실패 시 graceful skip
- 감성분석은 Claude API 호출이므로 비용 발생 — 기본 OFF, 명시적 옵션으로만 ON
- SQLite는 동시 쓰기 제한 있음 — 스케줄러가 단일 프로세스이므로 문제없지만, WAL 모드 권장
- `src/services/` 디렉토리가 없으면 생성 필요 (`__init__.py` 포함)
