## 기능 A: 1-24 — 시황 브리핑 cron 자동화

**목표**: generate_briefing.py를 오전 6시 / 오후 6시 KST에 자동 실행하도록 cron 등록

**컨텍스트**:
- `scripts/generate_briefing.py` — 이미 완성된 브리핑 생성기 (morning/evening)
- `scripts/run_briefing.sh` — cron용 래퍼 스크립트. **경로 버그 있음**: `/home/gint_pcd/projects/주식부자프로젝트` → `/home/window11/stock` 으로 수정 필요
- 텔레그램 봇 연결은 별도 PM이 담당 → 이 태스크는 **파일 출력까지만**
- 출력 디렉토리: `data/briefings/` (--save 플래그)

**작업 순서**:
1. `run_briefing.sh` 경로 수정 (`/home/gint_pcd/...` → `/home/window11/stock`)
2. `generate_briefing.py --schedule morning --save` dry-run 테스트 (실제 실행, 오류 확인)
3. cron 등록:
   - `0 21 * * * ...` (UTC 21:00 = KST 06:00) — morning 브리핑
   - `0 9 * * * ...` (UTC 09:00 = KST 18:00) — evening 브리핑
4. TASK.md 1-24 상태 → `진행` → 완료 시 `완료`
5. 커밋

**주의**: git push 금지. cron 등록은 `crontab -e` 대신 `crontab` 파이프로 처리.
