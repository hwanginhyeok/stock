# 주식부자프로젝트

> 한국/미국 주식 시장 분석 + AI 콘텐츠 생성 + SNS 자동 게시 플랫폼

## 세션 시작 규칙 (필수)

새 세션이 시작되면 코딩에 들어가기 전에 반드시 아래 순서를 따른다:

1. **TASK.md 확인** — `docs/프로젝트/TASK.md`에서 현재 상태 점검, 지연 항목 보고
2. **매뉴얼 로딩** — `.claude/rules/` 하위 파일 전체 읽기 (coding, workflow, architecture, git)
3. **데일리 미팅** — 진행 상황, 블로커, 이슈를 사용자와 공유
4. **방향성 논의** — 오늘 작업할 항목의 우선순위와 접근 방식을 사용자와 합의
5. **합의 후 착수** — 사용자가 동의한 작업만 진행

## 언어 규칙

- **코드**: 변수명, 함수명, 클래스명, 주석 → 영어
- **콘텐츠/문서**: 사용자 대면 콘텐츠, 문서, 커밋 메시지 → 한국어
- **설정 파일**: 키는 영어, 값은 용도에 따라 한국어/영어

## TASK 관리

- **통합 관리**: `docs/프로젝트/TASK.md` 단일 파일에서 전체 TASK + TODO 관리
- **테이블 형식**: `| # | 작업 | 중요도 | 담당 | 상태 | 비고 |`
- **상세 로그**: 설계 결정이 포함된 작업은 `docs/프로젝트/task/{TASK_ID}.md`에 기록
- **실시간 갱신**: 착수 시 → 진행, 완료 시 → 완료 섹션 이동, 새 작업 발견 시 즉시 등록

## 상세 규칙 (세션 시작 시 필수 로딩)

| 파일 | 내용 | 로딩 시점 |
|------|------|-----------|
| `.claude/rules/coding.md` | Python 컨벤션, 코드 패턴, 품질 체크리스트 | 세션 시작 |
| `.claude/rules/workflow.md` | 워크플로우 원칙, 보안 규칙, TASK 갱신 규칙 | 세션 시작 |
| `.claude/rules/architecture.md` | 디렉토리 구조, 에이전트 역할 | 세션 시작 |
| `.claude/rules/git.md` | Git 컨벤션 | 세션 시작 |

## 스킬 (반복 작업 시 참조)

| 파일 | 내용 | 트리거 |
|------|------|--------|
| `.claude/skills/README.md` | 스킬 작성 가이드 | 새 스킬 추가 시 |
