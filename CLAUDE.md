## 투자 철학 & 프로젝트 방향성

### 두 축 분리 원칙

이 프로젝트는 **두 가지 판단을 명확히 분리**한다. 이 둘을 섞으면 각각의 기준이 오염된다.

#### 축 1: 시장 흐름 읽기 — "언제 움직일까"
> 넘치는 데이터를 가시화하고, 거기서 시장의 흐름을 읽고 대응한다.

- **도구**: 유동성(FRED), FX, 섹터 ETF, 기술적 지표(RSI/ADX/MACD/Supertrend), 시장 레짐
- **산출물**: 모닝 이메일, 시그널 리포트 Excel, 지표 해석 체계
- **판단**: 지금 시장이 Risk-On인가 Risk-Off인가. 언제 포지션을 늘리고 줄일 것인가.

#### 축 2: 종목 선별 — "무엇을 살까"
> 종목 선별과 가치 판단은 독서와 배움을 통해 인사이트를 얻고, 그것을 토대로 결정한다.

- **도구**: 펀더멘탈 리서치, 아티클 집필, 장기 테제 구축
- **산출물**: 종목별 리서치 허브 (`data/research/stocks/`), X 아티클, 인스타 카드뉴스
- **판단**: 이 기업/자산의 방향성이 맞는가. 10년 뒤에도 보유할 수 있는가.

### 현재 집중 종목/자산
- **Tesla (TSLA)** — 수직통합 + 에너지 + 로봇 테제
- **Bitcoin (BTC)** — 디지털 금, 희소성 테제
- **Ethereum (ETH)** — 탈중앙화 플랫폼 테제

### 미완성 연결 고리 (향후 과제)
두 축을 연결하는 **의사결정 체계**가 아직 없다:
- 시장이 Risk-On → 선별된 종목에 언제, 얼마나 진입?
- 시장이 Risk-Off → 익스포저를 어떻게 줄일 것인가?
- 이 체계가 갖춰지면 진정한 시스템 트레이딩 완성.

## 세션 시작 규칙 (필수)

새 세션이 시작되면 코딩에 들어가기 전에 반드시 아래 순서를 따른다:

1. **TASK.md 확인** — `docs/프로젝트/TASK.md`에서 현재 상태 점검, 지연 항목 보고
2. **매뉴얼 로딩** — `.claude/rules/` 하위 파일 전체 읽기 (coding, workflow, architecture, git)
3. **데일리 미팅** — 진행 상황, 블로커, 이슈를 사용자와 공유
4. **방향성 논의** — 오늘 작업할 항목의 우선순위와 접근 방식을 사용자와 합의
5. **합의 후 착수** — 사용자가 동의한 작업만 진행

## 언어 규칙

- **코드**: 변수명, 함수명, 클래스명, 주석 → 영어
- **콘텐츠/문서**: 사용자 대면 콘텐츠, 문서, 커밋 메시지 → 한국어
- **설정 파일**: 키는 영어, 값은 용도에 따라 한국어/영어

## TASK 관리

- **통합 관리**: `docs/프로젝트/TASK.md` 단일 파일에서 전체 TASK + TODO 관리
- **번호 체계**: `{분야코드}-{순번}` — 1=시스템, 2=코드, 3=분석, 4=아티클, 5=리서치
- **상태**: `예정` → `요청` → `진행` → `완료`
- **갱신 규칙**: 상세 내용은 `.claude/rules/workflow.md` 참조
- **상세 로그**: 설계 결정이 포함된 작업은 `docs/프로젝트/task/{ID}.md`에 기록
